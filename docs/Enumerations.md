# cbang Enumerations

cbang's enumeration system replaces plain C++ `enum` with a typed
wrapper that round-trips between numeric value, name string, and
description.  The wrapper is generated by an X-Macro: define your
values once in a `.h`, include a `.def`, and you get
`parse(string)`, `toString()`, `isValid()`, iteration, JSON I/O,
streaming, and option-system integration — all for free.

Every status / state / type / level you see across cbang is built
with this: `cb::HTTP::Status`, `cb::Event::EventFlag`,
`cb::JSON::ValueType`, fah-client's `UnitState`, `CoreState`,
`ExitCode`, `RunResult`, etc.

## Concepts

- **`cb::Enumeration<T>`** — template that wraps a generated
  enumeration class.  Provides `toString`, `parse`, `isValid`,
  iteration, comparison.
- **`cb::EnumerationBase`** — non-templated base holding the
  numeric value.  Used by code that handles enums generically (the
  options system, JSON serialization).
- **X-Macro `.def` files** —
  `<cbang/enum/MakeEnumeration.def>` and
  `<cbang/enum/MakeEnumerationImpl.def>` do the generation work.
  Your enum files include them at the right point in conditional
  blocks.

The generated class is a `class`, not a C `enum`, so it has methods
and converts cleanly to a JSON string.  It also enums through the
constraint system, so you can clamp a config option to its values
with `cb::EnumConstraint<T>`.

## Defining an enumeration

Two files: a header and a tiny implementation.

### Header (`MyEnum.h`)

```cpp
#ifndef CBANG_ENUM
#ifndef CBANG_MY_ENUM_H
#define CBANG_MY_ENUM_H

#define CBANG_ENUM_NAME       MyEnum     // class name to generate
#define CBANG_ENUM_NAMESPACE  myapp      // namespace (optional)
#define CBANG_ENUM_PATH       myapp      // include path under src/
#define CBANG_ENUM_PREFIX     3          // strip this many chars from each name
#include <cbang/enum/MakeEnumeration.def>

#endif // CBANG_MY_ENUM_H
#else  // CBANG_ENUM

CBANG_ENUM(MY_RED)
CBANG_ENUM(MY_GREEN)
CBANG_ENUM(MY_BLUE)

#endif // CBANG_ENUM
```

The `#ifndef CBANG_ENUM` / `#else` split is the X-Macro pattern:
the file is `#include`'d *twice* — once to define the class
skeleton, once to expand the value list.  Your enum constants live
in only one place.

### Implementation (`MyEnum.cpp`)

```cpp
#define CBANG_ENUM_IMPL
#include "MyEnum.h"
#include <cbang/enum/MakeEnumerationImpl.def>
```

Three lines.  That's the entire `.cpp` file.

That's it — `myapp::MyEnum` is now usable.

## Value macros

Inside the `else` branch of the header, use any of:

| Macro | Effect |
|---|---|
| `CBANG_ENUM(NAME)` | Auto-numbered (sequential from 0 or previous + 1).  Description = `NAME`. |
| `CBANG_ENUM_VALUE(NAME, n)` | Explicit numeric value. |
| `CBANG_ENUM_DESC(NAME, "desc")` | Auto-numbered with custom description. |
| `CBANG_ENUM_VALUE_DESC(NAME, n, "desc")` | Both. |
| `CBANG_ENUM_ALIAS(NAME, OTHER)` | Make `NAME` an alias for `OTHER` (same numeric value, same string). |

Mix freely:

```cpp
CBANG_ENUM(LEVEL_RAW)
CBANG_ENUM_VALUE(LEVEL_ERROR,   1 << 0)
CBANG_ENUM_VALUE(LEVEL_WARNING, 1 << 1)
CBANG_ENUM_VALUE_DESC(LEVEL_INFO, 1 << 2, "informational")
CBANG_ENUM_ALIAS(LEVEL_LOG, LEVEL_INFO)
```

## The `CBANG_ENUM_PREFIX` setting

`CBANG_ENUM_PREFIX N` strips the first N characters from each
name when producing the string form.  Useful when your C++
identifiers are namespaced by a common prefix you don't want in
serialised output:

```cpp
#define CBANG_ENUM_PREFIX 5            // strip "UNIT_"

CBANG_ENUM(UNIT_ASSIGN)                // toString() returns "ASSIGN"
CBANG_ENUM(UNIT_DOWNLOAD)              // toString() returns "DOWNLOAD"
```

The C++ enum constants remain `UNIT_ASSIGN`, `UNIT_DOWNLOAD`, etc.
This is fah-client's pattern for `UnitState`, `CoreState`,
`ExitCode`.

If you do NOT set `CBANG_ENUM_PREFIX`, the full identifier is used
as the string.

## Using the generated class

```cpp
#include "MyEnum.h"

using namespace myapp;

MyEnum c   = MY_RED;                  // construct from enum_t
MyEnum c2  = MyEnum::parse("GREEN");  // parse from string
                                      // (after PREFIX stripping if set)

if (c.isValid())            ...;
std::cout << c.toString();            // -> "RED"
std::cout << c.getDescription();      // -> "RED" by default
std::cout << c;                       // streams toString()
unsigned n = c.toInteger();           // raw numeric value

if (c == MY_RED)            ...;
if (c == MyEnum::MY_BLUE)   ...;
```

Conversion to/from `enum_t` is implicit, so you can use a
`MyEnum` anywhere a plain enum would work — switch statements,
function args, comparisons.

### Iteration

```cpp
for (unsigned i = 0; i < MyEnum::getCount(); i++) {
  MyEnum v((MyEnum::enum_t)i);
  std::cout << v << " = " << v.toInteger() << '\n';
}
```

Useful for `--help` text or for validating an enum dropdown in a UI.

### Inheriting the enum

The common pattern is to inherit `Enum` into your own class so the
enum constants appear in your namespace:

```cpp
class Unit : public UnitState::Enum {
public:
  UnitState getState() const { ... }    // returns the typed wrapper
  void check() {
    if (getState() == UNIT_RUN) ...;    // bare constant works
  }
};
```

`UnitState::Enum` brings the constants into the class scope so you
don't have to write `UnitState::UNIT_RUN` everywhere.  fah-client
uses this on every class that needs to talk about a state machine.

## Integration with other cbang systems

### Options (`EnumConstraint`)

Restrict a command-line option to enum values:

```cpp
opts.add("level", "Verbosity level",
         new cb::EnumConstraint<MyLogLevel>())
    ->setDefault(MyLogLevel::LEVEL_INFO.toString());
```

The user gets a clear error like `Expected one of: RAW, ERROR,
WARNING, INFO, DEBUG` when they pass something invalid.

### JSON

Enum values serialise as their string form when written to a JSON
sink:

```cpp
sink.insert("state", unit.getState());     // -> "state": "RUN"
```

And `parse` will accept the string back:

```cpp
state = UnitState::parse(json.getString("state"));
```

This is the wire format for fah-client's WU state — see
`Unit::save` and `Unit::Unit(JSON)`.

### Streaming

`operator<<` is generated, so `std::cout << myenum;` works and gives
the string form.  Logger macros pick this up automatically:

```cpp
LOG_INFO(1, "State changed to " << state);    // emits "State changed to RUN"
```

### Switch statements

```cpp
switch (state) {
case UNIT_ASSIGN:   ...; break;
case UNIT_DOWNLOAD: ...; break;
...
}
```

Because the wrapper implicitly converts to `enum_t`, switch
statements compile cleanly and the compiler warns on missing
cases.

## Patterns

### Wire-protocol enums

When an enum is part of an external API (saved to disk, sent on the
wire, displayed in a UI), the *names* become part of the contract.
Renumbering is safe; renaming is not.  Document this in your
`TESTING.md` or equivalent (fah-client lists `UnitState` /
`CoreState` / `ExitCode` as wire-protocol artifacts).

### Bit flags

Use `CBANG_ENUM_VALUE` with powers of two:

```cpp
CBANG_ENUM_VALUE(EVENT_NONE,        0)
CBANG_ENUM_VALUE(EVENT_TIMEOUT, 1 << 0)
CBANG_ENUM_VALUE(EVENT_READ,    1 << 1)
CBANG_ENUM_VALUE(EVENT_WRITE,   1 << 2)
```

`toInteger()` returns the raw bitmask; combine with `|` on the
`enum_t` form.  `isValid()` only returns true for declared values,
so bitmasks-of-flags are *not* `isValid`.

### Aliases for renames

Keep external compatibility while renaming internally:

```cpp
CBANG_ENUM(DNS_A)
CBANG_ENUM_ALIAS(DNS_IPV4, DNS_A)         // same value; "IPV4" works for parse
```

### Tagging with descriptions

```cpp
CBANG_ENUM_DESC(HTTP_BAD_REQUEST, "Bad Request")
CBANG_ENUM_DESC(HTTP_FORBIDDEN,   "Forbidden")
```

`getDescription()` returns the friendly form; `toString()` returns
the identifier.  Used by `cb::HTTP::Status` so error responses can
emit `400 Bad Request` rather than `400 BAD_REQUEST`.

## Common pitfalls

- **Forgetting the `.cpp` file.**  The header alone leaves linker
  errors complaining about missing `toString`/`parse` symbols.
- **Putting the file outside `cbang/enum/`.**  Set
  `CBANG_ENUM_PATH` to your subdirectory; otherwise the include
  resolves wrong.
- **Renaming an enum constant after it ships.**  Any persisted
  state or wire message using the old name now fails to parse.  Use
  `CBANG_ENUM_ALIAS` to keep the old name working.
- **Confusing `toString()` and `getDescription()`.**  `toString`
  is the (prefix-stripped) identifier — stable, machine-readable.
  `getDescription` is human-readable.
- **Comparing `MyEnum` to a raw int.**  Compare to the enum
  constant or wrap the int in the typed form first.
- **Using the wrong PREFIX value.**  `CBANG_ENUM_PREFIX 5` strips
  five chars; mismatched with your identifier prefix gives garbled
  string forms.
